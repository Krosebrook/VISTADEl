# Contributing to VISTADEL

Thanks for considering contributing to VISTADEL! This framework is built on real client work and community feedback.

> **Full contribution guide:** See [`vistadel-complete/CONTRIBUTING.md`](./vistadel-complete/CONTRIBUTING.md) for the complete contribution guidelines, writing standards, quality gates, and PR process.

---

## Quick Start

1. **Fork** the repository
2. **Create a branch:** `feature/your-feature` or `fix/your-fix`
3. **Make changes** following the [contribution guide](./vistadel-complete/CONTRIBUTING.md)
4. **Test** your changes (run the server with `npm start` and verify affected routes)
5. **Submit a PR** using the pull request template

---

## Types of Contributions

| Type | Where to start |
|------|---------------|
| Fix a typo or clarity issue | Open a PR directly |
| Add a new example | Open an issue first |
| New phase content or tools | Open a GitHub Discussion |
| Bug reports | Create a GitHub Issue |
| Ideas & questions | GitHub Discussions |

---

## Quality Bar

Every PR must include real examples, tested code, and explained tradeoffs. Theory-only PRs will be returned for revision.

---

## Questions?

- **Contribution questions:** Open a GitHub Discussion
- **Content questions:** Comment on the relevant issue
- **INT-specific:** contact@intinc.com
