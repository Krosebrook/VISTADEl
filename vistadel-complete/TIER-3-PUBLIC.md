# TIER 3: Public Community

## What is Tier 3?
Open-source framework for founders, engineers, and teams shipping apps.

**Use case:** "Here's how to ship fast with modern tools. Here's the framework we use."

**Audience:** Public community, developers, founders, anyone interested in app development

## Tone (Public Tier)

**Goal:** Educational, practical, honest, community-first.

**Examples:**
- ❌ "VISTADEL is the only framework you need"
- ✅ "VISTADEL is a framework we've tested across 50+ projects. Here's what works."
- ❌ "If you use Claude, you'll build 10x faster guaranteed"
- ✅ "Claude + thoughtful prompts have helped us move faster on these specific tasks"
- ❌ "Supabase is the only database you should use"
- ✅ "We use Supabase because [reasons]. Here's how it compares to [alternative]."

## Community Standards

### Quality Gates
- Every code example must be tested
- Every decision must have a tradeoff statement
- Every recommendation must be backed by evidence
- Every tool must be maintained (not abandoned)

### Contribution Requirements
- PRs must include examples
- No theory-only recommendations
- Explain why, not just how
- Test before submitting

### Attribution
- Credit INT Inc. for framework
- Credit individual contributors
- Credit community improvements
- Link to original case studies

## Updating Tier 3

When core phases change:
1. Update public versions
2. Get community feedback via GitHub Discussions
3. Accept PRs for improvements, additions
4. Maintain changelog of updates
5. Announce major changes in blog
